# ARQuest - AR Scavenger Hunt Implementation Guide

## Overview

This document describes the complete implementation of the ARQuest Flutter application, an AR-based scavenger hunt game. The app uses Firebase, Google Maps, geolocator, and AR capabilities to provide an immersive treasure-hunting experience.

## Architecture

### Project Structure

```
lib/
├── main.dart                      # Firebase initialization & app entry point
├── firebase_options.dart          # Firebase configuration (generated by FlutterFire CLI)
├── models/
│   └── quest.dart                 # Quest and CapturedTreasure data models
├── services/
│   ├── firestore_service.dart     # Firestore operations & authentication
│   ├── location_service.dart      # Geolocator & proximity detection
│   └── photo_service.dart         # Camera & Base64 photo handling
└── screens/
    ├── map_screen.dart            # Google Maps with quest markers
    └── ar_view_screen.dart        # AR view with proximity indicator
```

## Features Implemented

### 1. Firebase Initialization
**File**: [main.dart](lib/main.dart)

- Initializes Firebase with platform-specific configuration via `firebase_options.dart`
- Enables anonymous authentication for users
- Uses Firestore for quest and treasure data storage

### 2. Map Screen
**File**: [screens/map_screen.dart](lib/screens/map_screen.dart)

- Displays Google Maps with user's current location
- Shows markers for all quests fetched from `/quests` collection
- Real-time location tracking for continuous position updates
- Tap markers to navigate to AR view
- Requests and validates location permissions

### 3. Proximity Detection Logic
**File**: [services/location_service.dart](lib/services/location_service.dart)

- Monitors user location in real-time (via geolocator)
- Calculates distance to quest markers using Haversine formula
- Triggers "Enter AR Hunt" button when user is within 20 meters
- Shows distance and proximity indicator

### 4. AR View Screen
**File**: [screens/ar_view_screen.dart](lib/screens/ar_view_screen.dart)

- Displays AR view with 3D model placeholder (ready for model integration)
- Shows real-time distance to quest
- Visual proximity indicator with progress bar
- "Capture & Save" button enabled only when within 20m range
- Loads 3D .glb models from `modelUrl` field in Firestore

### 5. Photo Capture & Base64 Encoding
**File**: [services/photo_service.dart](lib/services/photo_service.dart)

- Initializes device camera (rear camera preferred)
- Captures photos when user taps AR object
- Converts images to Base64 strings (Spark Plan compliant)
- Supports image resizing for optimal Firestore storage
- Provides camera disposal and cleanup

### 6. Firestore Integration
**File**: [services/firestore_service.dart](lib/services/firestore_service.dart)

**Collections**:
- `/quests` - Quest data with geolocation and model URLs
- `/captured_treasures` - User submissions with Base64 images

**Operations**:
- Real-time quest stream listening
- Single quest retrieval
- Captured treasure submission with metadata
- User-specific treasure history
- Anonymous authentication support

### 7. Data Models
**File**: [models/quest.dart](lib/models/quest.dart)

```dart
class Quest {
  String id;           // Document ID
  String name;         // Quest name
  double lat, lng;     // Geographic coordinates
  String modelUrl;     // 3D model URL (.glb)
  String description;  // Quest description
  String? imageUrl;    // Optional thumbnail
}

class CapturedTreasure {
  String id;              // Document ID
  String userId;          // User ID (anonymous or authenticated)
  String questId;         // Associated quest
  String base64Image;     // Base64-encoded photo
  DateTime timestamp;     // Capture time
}
```

## Configuration Required

### 1. Firebase Project Setup

1. Go to [Firebase Console](https://console.firebase.google.com)
2. Create a new project or use existing one
3. Enable **Firestore Database** (Spark Plan)
4. Enable **Anonymous Authentication**
5. Download `google-services.json` from Firebase Console
6. Verify it's placed at `android/app/google-services.json` ✓

### 2. Update firebase_options.dart

Run FlutterFire CLI to generate platform-specific configuration:

```bash
flutterfire configure --project=YOUR_PROJECT_ID
```

This will automatically populate [firebase_options.dart](lib/firebase_options.dart) with correct credentials.

Alternatively, manually update the placeholder values in `firebase_options.dart`:
- Replace `YOUR_PROJECT_ID` with your Firebase project ID
- Replace API keys from Firebase Console

### 3. Firestore Collections Setup

Create the following collections in Firestore:

**Collection: `/quests`**
```json
{
  "id": "quest_001",
  "name": "Hidden Treasure",
  "lat": 37.7749,
  "lng": -122.4194,
  "modelUrl": "https://example.com/treasure.glb",
  "description": "Find the hidden treasure in the park",
  "imageUrl": "https://example.com/preview.jpg"
}
```

**Collection: `/captured_treasures`**
(Auto-created by app when photos are captured)
```json
{
  "userId": "anonymous_user_id",
  "questId": "quest_001",
  "base64Image": "data:image/jpeg;base64,...",
  "timestamp": "2024-01-15T10:30:00Z"
}
```

## Implementation Notes

### Spark Plan Compliance

- ✅ **No Firebase Storage** - All images stored as Base64 in Firestore
- ✅ **Firestore only** - Uses Firestore for all data (no Realtime Database)
- ✅ **Anonymous auth** - Users can use app without account creation
- ⚠️ **Document size limit** - Base64 images must be <1MB per Firestore document

### Image Size Optimization

The `PhotoService` includes image resizing to reduce Base64 payload:

```dart
// Resize to 800px width before encoding
final resized = await photoService.resizeImage(imageBytes, maxWidth: 800);
final base64 = base64Encode(resized);
```

### Real-time Updates

- **Quests**: Firestore stream listener updates markers in real-time
- **Location**: Geolocator stream provides updates every 1 second
- **Treasures**: Auto-saved with server timestamp

## Troubleshooting

### "Location permission denied"
- Check Android permissions are declared in AndroidManifest.xml
- Grant app permissions in device settings
- Restart app after granting permissions

### "No cameras available"
- Ensure device has rear camera
- Test on physical device (emulator may not have camera)

### "Firebase not initialized"
- Verify `google-services.json` is in `android/app/`
- Run `flutterfire configure` if firebase_options.dart is empty
- Check Firebase project ID matches

### "Firestore connection timeout"
- Verify internet connection
- Check firestore_service.dart error logs
- Ensure Firestore database is enabled in Firebase Console
